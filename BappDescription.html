SAVER_LOGGER is a professional-grade Burp Suite extension designed for comprehensive HTTP request logging from all Burp Suite tools, including the Proxy, Scanner, Repeater, Intruder, and others. Combined with the metadata capturing abilities, SAVER_LOGGER offers a complete front to back logging solution inclusive of automatic or manual CSV data export, thereby providing a verifiable proof of authenticity.

Features

Core Functionality

  • Comprehensive Logging: Fully captures every HTTP request from all Burp tools
  • Complete Metadata: Records the host, method, URL, status code, tool source, insertion points, and timing
  • Insertion Point Detection: Works with the following tools -
    • Intruder: Counts ? payload markers
    • Scanner: Counts tested parameters
    • Other tools: Identifies available parameters
  • Request Counting: Tracks the number of times each URL has been accessed
  • Timing Analysis: Documents start and end timestamps for each request

Export & Backup

  • Auto-backup on Exit: Offers a crash-safe backup with timestamped filename, for instance, SAVER_LOGGER_AUTOSAVE_DDMMYYYY_HHMMSS.csv
  • Periodic Auto-backup: A user-configurable interval (10-3600 seconds) to a custom folder
  • Manual Export: A user-triggered CSV export with a file chooser dialog
  • Manual Backup: An immediate timestamped backup, like SAVER_LOGGER_BACKUP_DDMMYYYY_HHMMSS.csv
  • Single File Approach: A run-time configurable approach where periodic backups overwrite the same file to prevent fragmentation

Data Integrity

  • UTF-8 Encoding: Ensures complete international character support
  • CSV Sanitization: Invariably manages commas, newlines, and special characters
  • Footer Metadata: Includes Burp version, export timestamp, and unique runtime ID
  • Session Tracking: A unique session identifier for complete audit trails

User Interface

  • Dashboard Tab: Allows quick access to export, backup, and clear functions
  • Settings Tab: Allows users to set the backup folder, interval, and enable/disable auto-backup
  • Status Area: Gives real-time logging of all operations with timestamps
  • Confirmation Dialogs: Gives intuitive feedback for all user actions

Usage

Once the extension is activated, logging occurs automatically for all HTTP traffic. More fine-grained control is available through the SAVER_LOGGER Dashboard Tab, where users can navigate to the Settings tab to specify backup behavior. :

Dashboard Actions

  • Export CSV: This option allows users to choose the filename and location. By default, the name is SAVER_LOGGER_Export_YYYYMMDD_HHMMSS.csv. All logged requests are exported with complete metadata.
  • Backup Now: Creates an instant timestamped backup that saves to the configured backup folder. The filename will be SAVER_LOGGER_BACKUP_DDMMYYYY_HHMMSS.csv. A dialog providing confirmation and file path will be shown.
  • Clear Logs: Removes all recorded requests from memory, depending on user confirmation. Previously exported files are not deleted.

Settings Configuration

  • Auto-Backup Settings:
    • Enable/Disable: Allows the user to toggle the feature of automatic periodic backups
    • Backup Interval: Enables the user to adjust the time between auto-backups within a range of 10-3600 seconds
    • Backup Folder: Permits the user to select a directory for backups
    • Default Location: Sets the destination to the Desktop

Naming Conventions

Different types of backups and exports will have different filename conventions as follows:

  • Periodic Auto-backup → SAVER_LOGGER_AUTOSAVE.csv
  • Exit/Crash Backup → SAVER_LOGGER_AUTOSAVE_DDMMYYYY_HHMMSS.csv
  • Manual Backup → SAVER_LOGGER_BACKUP_DDMMYYYY_HHMMSS.csv
  • Manual Export → SAVER_LOGGER_Export_YYYYMMDD_HHMMSS.csv
